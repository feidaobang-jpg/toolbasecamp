# Main-site i18n (toolbasecamp.com)

Subdomains **pdf** / **dev** have their own i18n — do not duplicate here.

## Languages

- **English** (`en`) — default for non-Chinese browsers
- **简体中文** (`zh-CN`) — auto-selected when `navigator.language` starts with `zh`

Users can switch via the **中文 / EN** control in the header. Choice is stored in `localStorage` key `tb-locale`.

## Adding UI text (required for new features)

1. Add the **same key** to both:
   - `public/js/locales/en.js`
   - `public/js/locales/zh-CN.js`
2. Use the key in HTML or JS — never hardcode user-facing strings on the main site.

### HTML (static text)

```html
<script src="js/locales/en.js"></script>
<script src="js/locales/zh-CN.js"></script>
<script src="js/i18n.js"></script>
<!-- then config.js, common_ui.js, etc. -->

<h1 data-i18n="guestbook.title">Guestbook</h1>
<input data-i18n-placeholder="guestbook.messagePlaceholder" />
```

Subdirectory pages use `../../js/locales/...`.

### JavaScript (dynamic text)

```javascript
var label = t('guestbook.posted');
```

With parameters: `t('tools.jsonToJava.parseError', { message: err.message })`

### Config-driven labels (`config.js`)

Use `nameKey` / `titleKey` / `descriptionKey` / `ctaKey` instead of literal `name` / `title`:

```javascript
{ nameKey: 'nav.tools', url: 'index.html' }
{ titleKey: 'tools.pdfToWord.title', url: 'html/docs/pdf-to-word.html' }
```

Resolve in JS with `tbLabel(item)` or `t(item.titleKey)`.

## Key naming

Use dot-separated paths, grouped by area:

| Prefix | Example |
|--------|---------|
| `nav.*` | `nav.tools` |
| `site.*` | `site.description` |
| `tools.<toolId>.*` | `tools.pdfToWord.convert` |
| `tools.groups.*` | `tools.groups.document` |
| `guestbook.*` | `guestbook.post` |
| `auth.*` | `auth.login` |
| `common.*` | `common.loading` |

## Locale change event

When the user switches language, components can refresh:

```javascript
document.addEventListener('tb:locale', function () {
  tbApplyI18n(document);  // re-apply data-i18n attributes
  renderMyDynamicUi();
});
```

`base.js`, `tools-hub.js`, `guestbook.js`, and `common_ui.js` already listen for this event.

## Checklist for new tools

- [ ] Keys in `en.js` and `zh-CN.js`
- [ ] `titleKey` in `toolsConfig` (`config.js`)
- [ ] Tool page loads `locales/en.js`, `locales/zh-CN.js`, `i18n.js` before `config.js`
- [ ] Visible HTML uses `data-i18n` / `data-i18n-placeholder`
- [ ] JS toasts/errors use `t('...')`
